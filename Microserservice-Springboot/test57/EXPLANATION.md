# Test 57: Functional Web (Router Functions)

An alternative to `@RestController` for defining APIs.

## Key Files:
- `FunctionalConfig.java`: Shows how to define routes without controllers.
- `EXPLANATION.md`: Why explicit routing can be cleaner than discovery-based controllers.

## Concept:
Instead of putting annotations on classes, you define your API "Map" in one central place. It's more functional, easier to test, and has zero reflection overhead.
