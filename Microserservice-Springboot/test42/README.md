# Test 42: gRPC (High Performance)

Binary communication for super-fast microservices.

## REST (Text/JSON) vs gRPC (Binary):
- **REST**: Like sending a letter in a huge envelope with lots of extra paper. 
- **gRPC**: Like sending a secret code. It's much smaller and 10x faster.

## Why use it?
For **Internal Communication** between services where speed is critical. You use Protocol Buffers (`.proto` files) to define your data.

It's developed by Google for massive scale!
