# Test 35: KISS (Keep It Simple, Stupid) Principle

## 👶 1. What is KISS? (Simple Explanation)
KISS means **"Most systems work best if they are kept simple rather than made complicated."**
Imagine you want to **Hang a Picture** on a wall.
- **Complex**: Buying a laser-guided drilling robot, scanning the wall for pipes, and using a titanium bolt.
- **Simple**: Using a small hammer and a nail.

---

## 🖱️ 2. Coding Application
Many developers (especially seniors!) like to use complex "Design Patterns" where a simple `if` statement would work.

### Bad Code:
- Using a `Factory`, a `Strategy`, and an `Observer` just to print "Hello Vikas."
- Writing one-line code that is 500 characters long using "Stream API" magic that nobody can read.

### Good Code:
- Clean, boring code that a junior can understand in 5 seconds.
- Code that uses standard names and easy-to-follow logic.

## 🚀 3. Master's Rule:
"Debugging is twice as hard as writing the code in the first place. Therefore, if you write the code as cleverly as possible, you are, by definition, not smart enough to debug it."
