# Test 36: YAGNI (You Ain't Gonna Need It)

## 🔮 1. What is YAGNI? (Simple Explanation)
YAGNI means **"Do not add functionality until it is necessary."**
Imagine you are building a **Small Shed** in your garden.
- You think: "Maybe one day I will have a helicopter, let's build a helipad on the roof!"
- You spend months building the helipad. 
- You never buy a helicopter. 
You wasted time, money, and now your shed roof is too heavy and might collapse.

---

## 🖱️ 2. Coding Application
- Don't build a "Multi-language Support" system if your app only has 10 users in Maharashtra.
- Don't add a "Complex Caching Layer" before you even know if your app is slow.
- Don't write generic code that handles "Every possible future scenario."

### The Problem with over-planning:
1. **Wasted Time**: You build things that never get used.
2. **Maintenance**: You have to fix bugs in code that isn't even part of the current requirements.
3. **Clutter**: The real logic gets hidden under piles of "future-proofing."

## 🚀 3. Master's Rule:
Build for today, but design for change. Write clean code so that adding the "Helipad" later is easy, but don't build it yet!
