# Test 61: gRPC (High-speed Microservices)

Make service-to-service calls at the speed of light.

## Why use gRPC?
- **HTTP/2**: Multiplexing and binary transport.
- **Protocol Buffers**: Much faster than JSON.
- **Bi-directional Streaming**: Server and client can talk at the same time.

It's primarily used for **Internal Microservices** communication because it's much faster than REST but harder for browsers to consume directly.
