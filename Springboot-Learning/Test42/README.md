# Test 42: Spring Boot with Messaging (Kafka)

The backbone of event-driven architecture.

## Kafka vs HTTP:
- **HTTP**: Synchronous (Wait for response). User is blocked.
- **Kafka**: Asynchronous (Send and forget). High throughput.

## Use Cases:
1. **Activity Tracking**: Logging user clicks.
2. **Order Processing**: Send order to warehouse, billing, and shipping services at once.
3. **Real-time streams**: Finance data or social media feeds.
